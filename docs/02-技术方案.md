# 02 - 技术方案

## 技术选型

| 层面 | 选择 | 原因 |
|------|------|------|
| **应用形式** | PWA（渐进式Web应用） | 无需安装APK，浏览器打开即用，支持添加到主屏幕 |
| **前端技术** | 纯HTML + CSS + JavaScript | 零依赖，体积小，加载快，无需学习框架 |
| **数据存储** | IndexedDB | 浏览器内置数据库，支持离线、容量大、支持索引查询 |
| **离线支持** | Service Worker | PWA标配，缓存静态资源，断网也能打开 |
| **部署** | GitHub Pages | 免费、自动HTTPS、全球CDN |

## 为什么不用框架？

- 用户是普通用户，不需要复杂工程化
- 单文件开发，方便管理和部署
- 体积极小（< 50KB），秒开
- 不依赖npm/webpack等工具链

## 架构设计

```
┌──────────────────────────────────┐
│           index.html             │
│  ┌────────────┐ ┌─────────────┐  │
│  │   UI层      │ │   数据层     │  │
│  │ (HTML/CSS)  │ │  (JS/IDB)   │  │
│  │             │ │             │  │
│  │ 今日页      │ │ CRUD操作    │  │
│  │ 统计页      │ │ 查询聚合    │  │
│  │ 记录页      │ │             │  │
│  │ 记账弹窗    │ │             │  │
│  └────────────┘ └─────────────┘  │
│              ↕                    │
│         IndexedDB                │
└──────────────────────────────────┘
         ↕ (Service Worker)
    ┌──────────┐
    │  sw.js   │  ← 缓存静态资源
    └──────────┘
    ┌──────────────┐
    │ manifest.json │  ← PWA配置
    └──────────────┘
```

## 浏览器兼容性

- Chrome for Android 80+
- Edge for Android 80+
- Samsung Internet 13+
- 覆盖绝大多数安卓设备

## 数据存储方案

使用 IndexedDB，数据库名称：`XiaoDianDB`

- 数据库版本：1
- 对象仓库：`transactions`
- 索引：`date`（用于按日期查询和排序）
